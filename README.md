# IS-23
Учебный репозиторий для студентов группы ИС-23
1.Настройка конфигурации git:
    git config --list - посмотреть список настроек
    git config user.name - посмотреть значение определенной настройки
    git config --global user.name "John Doe" - установить значение настройки (--global - для всех пользователей)
2. Получение помощи git:
    git help (имя команды) - окроет вкладку в браузере с описанием команды
    git (имя команды) -h - даст помощь в командной строке

    ***Работа с локальным репозиторием***
3. Создание локального репозитория:
    cd C:/Users/user/my_project - переходим в проект
    git init - иницилизируем локальный git репозиторий
    ИЛИ
    git clone https://github.com/libgit2/libgit2 - клонирование удаленного репозитория так же создаст локальный репо.
    git clone https://github.com/libgit2/libgit2 (имя репо) - задание имени репозитория, отличающегося от текущего.
4. Запись изменений в репозиторий (индексация):
    git status - проверка изменений в проекте
    git status -s (сокр. short) - компактная запись изменений
    git add (имя файла) - добавить файл в отслеживание ГИТом
    git add * - добавить все файлы в отслеживание ГИТом.
    git restore (имя файла) - восстановить файл до того состояния, в котором он был на момент добавления в отслеживание
5. Игнорирование ГИТом файлов:
    Создание в корне проекта файла .gitignore и запись в него различных шаблонов и паттернов. Файлы, которые попадут под эти шаблоны - отслеживаться не будут. Для поиска шаблона юзай веб или gpt.
6. Просмотр изменений:
    Для того, что бы посмотреть изменения до индексации (git add), используем команду git diff.
    Для просмотра уже зафиксированных изменений, которые попадут в коммит, используем команду git diff --staged | git diff --cached (синонимы)
7. Фиксация изменений - коммит.
    Для фиксации индексированных изменений, выполняем команду git commit -m 'сообщение комита'
    Для автоматической индексации (git add) можно добавить флаг -a: git commit -a -m 'сообщение'.
8. Удаление файлов, добавленных в индексацию.
        git rm --cached (имя файла) - удалить отслиживание ГИТом. (убрать из индеса)
        git rm (имя файла) - полностью удалить файл
9. Просмотр изменений проекта (просмотр коммитов)
    git log - выводит построчно историю коммитов
    git log -p(patch, показать изменения) -2(два последних коммита)
    git log --pretty=oneline - вывести комииты в одну строку (удобно читать)
    git log --pretty=format:"%h" - сокращенный хеш комита (все флаги смотрите в интернете)
10. Операции отмены:
    Если вы забыли что либо внести в комит - не стоит создавать новый комит, так как это засоряет историю и делает ее менее читабельной. Для внесения правок в текущий комит, вы можете использовать команду git commit --amend и внести новое сообщение комита. Индексированные файлы будут добавлены в текущий комит.
    git restore (имя файла) - отменить изменения в файле.
    git restore --staged (имя файла) - отменить индексацию файла
    ***Работа с удаленными репозиториями***
11. Добавление удаленных репозиториев:
    git remote - показать список подключенных удаленных репозиториев
    git remote show (имя репо) - получение больше ифно о репо
    git remote add (url) (псевдоним) - добавление репозитория под указанным псевдонимом (если не указать - origin по умолчанию)
12. Получение данных с удаленных репо:
    git fetch (имя репо) - подтянуть все комиты и ветки в ваш локальный репо (не трогает ваши наработки)
13. Отправка изменений в удаленный репо:
    git push (имя репо) (имя ветки) - отправка измененией в ветку в репо
14. Удаление и переименование репо:
    git remote rename (старое имя) (новое имя) - переименует репо
    git remote remove (имя репо) - удалить репо
    ***Теги***
15. Добавление, удаление и просмотр тегов:
    Просмотр тегов git tag
    Детальный просмотр тега git show (тег)
    Добавление аннотированного тега (тег, который содержит расширенную ифнормацию) git tag -a (версия) -m (сообщение)
    Добавление легковесного тега git tag (имя тега)
    При обычно пуше теги не переносятся, это надо делать отдельно - отпрвка тега на удаленый репо git push (имя) (тег)
    Удаление тего - git tag -d (имя тега)
    При удалении тега он автовтоматически не удаляется из удаленного репо - для удаления нужна команда 
    git push origin --delete (имя тега)
    ***Ветвление в git***
16. Создание,переключение и удаление веток:
    git branch - посмотреть список локальных веток
    git branch -v посмотреть последний комит на каждой ветке
    git branch --merged - посмотреть, какие ветки уже влиты в текущую ветку
    git branch --no-merged - узнать, какие ветки еще не влиты в текущую
    git branch --move (старое имя ветки) (новое имя ветки) - переименовать локальную ветку
    git push --set-upstream origin (новое имя ветки(такое же как и локальное)) - отправить ветку в удаленный репо
    git push origin --delete (старое имя ветки) - удалить старую ветку из удаленного репо
    git branch (имя ветки) - создание новой ветки с именем
    git checkout (имя ветки) - переключится на указанную ветку
    git checkout -b (имя ветки) - создание и переключение на ветку
    git switch (имя ветки) - переключится на ветку (новый функционал)
    git switch -c (имя ветки) - создать и переключится
    git switch (-) - вернутся на ветку назад
    git branch -d (имя ветки) - удалить ветку
    git branch -D (имя ветки) - удалить ветку в которой есть изменения
17. Слияние веток:
    Из ветки, в которую нужно влить другую ветку, выполнить команду git merge (имя вливаемой ветки)
    Основной конфликт слияния - когда вносятся изменения в один и тот же файл на разных ветках. При конфликте нужно решить, какие изменения оставить - внести их в индекс и сделать коммит.
18. Отправка изменений:
    Если вы работаете совместно, и кто то до вас уже отправил изменения в удаленный репо, то git отклонит ваш push. Вам нужно сначала подтянуть все свежие измпенения с удаленного репо, а затем только отправлять свои.
